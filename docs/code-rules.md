# Правила кода

## Один модуль — одна зона ответственности

- **Одна функция = одна задача.** Если функция делает и расчёт, и сохранение, и отрисовку — разбить на части.
- **Не дублировать код.** Повторяющиеся куски (обновление баланса в шапке, нормализация числа) выносить в `utils.js` или в сервисы.

## Стиль и явность

- **Не использовать глобальные переменные** для состояния приложения (кроме принятого экспорта `db` из `core/db.js` как единственного хранилища).
- **Имена функций** — глаголы и понятные сущности: `calcBalance`, `distributeGlobalIncome`, `renderUsers`, `refreshTotalBalance`.
- **Без магии:** явные импорты, явные параметры, без скрытых побочных эффектов вне описанных слоёв (см. `architecture.md`).

## Безопасность и данные

- Пользовательский текст в DOM — через `textContent` / `createTextNode`, не через `innerHTML` с неэкранированными строками.
- Числа из формы и из ответов API — через `safeNumber` / `parseMoneyInput` / `roundMoney` где уместно.

## Структура файлов

Не размещать бизнес-логику в `ui/`. Не размещать работу с DOM в `services/`. Не дублировать `db` в нескольких модулях.
