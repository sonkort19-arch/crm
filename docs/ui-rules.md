# Правила UI

## Роль слоя UI

- **UI только отображает** состояние из `db` и результаты вызовов сервисов/утилит (например, `calcBalance` для отображения — это чтение, не бизнес-правило создания операции).
- **В `render` не должно быть расчётов новых операций** (нет «самовольных» начислений внутри `renderUsers` / `renderDetails`). Суммы к добавлению в лог определяются только в `services/`.

## Авторизация и роли

- Экран входа: `#loginScreen`; основное приложение: `#appShell` (см. `authView.js`, `index.html`).
- Элементы только для администратора помечаются **`data-auth="admin"`**; видимость задаётся в `applyRoleVisibility()` (`render.js`).
- Сотрудник (`employee`) не видит настройки, глобальное распределение, добавление сотрудников; сервер при этом всё равно проверяет права на API.
- Кнопка **«Выйти»** вызывает `logout()` и перезагрузку страницы.

## Поток изменений

- **Все изменения данных** идут через **services** (HTTP к API с Bearer). События в `events.js` вызывают функции сервиса, затем обновляют экран через `render*.js` или `refreshTotalBalance`.

## DOM

- События — в `events.js` (и при необходимости локальные обработчики в `renderSettings`, создаваемые при отрисовке полей).
- Состояние кнопок (disabled) — в `actionButtons.js`, без бизнес-логики.

## Форма и id

- Соответствие `id` в `index.html` и в `getElementById` — проверять при изменениях (см. `dev-checklist.md`).
