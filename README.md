# Too CRM

Статическое веб-приложение (HTML / CSS / JS), без серверного бэкенда. Данные и тема хранятся в `localStorage` в браузере.

## Деплой на GitHub → Render

1. Убедитесь, что в коммит **не** попадает папка `node_modules/` (она в `.gitignore`). Для работы сайта достаточно файлов в корне: `index.html`, `app.js`, `styles.css`, `manrope.css`, `percent-logic.js` и т.д.
2. Подключите этот каталог к репозиторию **`sonkort19-arch/crm`**, ветка **`main`** (например: склонировать репозиторий, скопировать сюда файлы проекта поверх, затем `git add`, `commit`, `push` **или** инициализировать `git` здесь и задать `origin` на тот же URL).
3. На **Render** для сервиса типа **Static Site**: **Publish directory** = `.` (корень), **Build command** можно оставить пустым или `echo "skip"`, если требуется непустая команда. После пуша в `main` сработает автодеплой; проверьте логи и URL сервиса (например `https://crm-q8v5.onrender.com`).

Повторные обновления: изменения → `git commit` → `git push` → дождаться окончания деплоя на Render.

## Зависимости npm (необязательно)

В корне есть `package.json` с `@fontsource/manrope`. Для открытия сайта из файла шрифт уже подключён через **`manrope.css`**. `npm install` нужен только если вы сознательно переходите на импорт из `@fontsource`.

## Безопасность

Логины и пароли для входа заданы в клиентском коде (`app.js`) и **видны любому**, кто откроет исходники страницы. Для публичного или чувствительного сценария со временем потребуется серверная авторизация. В рамках текущей архитектуры это ограничение осознанное.
